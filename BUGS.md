# BUGS.md
Найдены несоответствия между документацией / Postman-коллекцией и фактическим поведением API https://qa-internship.avito.com.

---

## 1. Некорректный формат ответа при создании объявления
### Ожидалось (по документации):
{
  "id": "<string>",
  "sellerId": "<integer>",
  "name": "<string>",
  "price": "<integer>",
  "statistics": {...},
  "createdAt": "<string>"
}

### Фактически возвращается:
{
  "status": "Сохранили объявление - <UUID>"
}

➡ НЕТ:
- id
- sellerId
- name
- price
- statistics
- createdAt

---

## 2. Поле statistics.likes не принимает значение 0
### Ожидалось:
likes = 0 — валидно

### Фактически:
400 Bad Request  
"поле likes обязательно"

➡ API ошибочно валидирует значение 0 как отсутствие поля.

---

## 3. Цена price=0 не принимается
### Ожидалось:
price может быть 0

### Фактически:
400 Bad Request

➡ Документация противоречит реальному API.

---

## 4. Цена price может быть отрицательной
### Ожидалось:
price < 0 → ошибка 400

### Фактически:
API возвращает 200 и создаёт объявление.

➡ Нет валидации отрицательных цен.

---

## 5. Имя name длиной >255 символов не вызывает ошибку
### Ожидалось:
Ошибка валидации

### Фактически:
API принимает любые строки.

---

## 6. sellerID = -1 считается валидным
### Ожидалось:
sellerID должен быть положительным целым числом

### Фактически:
GET /api/1/-1/item возвращает 200 OK

➡ Нет проверки валидности sellerID.

---

## 7. Статистика по умолчанию не соответствует документации
### Ожидалось:
API самостоятельно заполняет данные статистики

### Фактически:
GET statistic возвращает массив из объектов, но документация ожидает другой формат.
